# Backend Documentation

This backend service is designed to scrape, store, and analyze ticket data from Vivid Seats API to provide real-time analytics and insights.

## Documentation Files

- **SYSTEM_DESIGN.md** - Overall architecture and design decisions
- **PSEUDO_CODE.md** - Pseudo code for key algorithms and processes
- **DATABASE_SCHEMA.md** - Database structure and relationships
- **SUPABASE_SETUP.md** - Supabase configuration guide
- **API_DOCUMENTATION.md** - API endpoints and usage

## Getting Started

1. Review SYSTEM_DESIGN.md for architecture overview
2. Follow SUPABASE_SETUP.md to configure your database
3. See API_DOCUMENTATION.md for endpoint specifications
